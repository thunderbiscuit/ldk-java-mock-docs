      • get_channel_type

        @Nullable() ChannelTypeFeatures get_channel_type()

        The channel type that this channel will represent. If none is set, we derive the channel type from the intersection of our feature bits with our counterparty's feature bits from the Init message. This is required to match the equivalent field in [`OpenChannel::channel_type`]. Note that the return value (or a relevant inner pointer) may be NULL or all-0s to represent None

      • set_channel_type
